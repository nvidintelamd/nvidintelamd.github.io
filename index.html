1. URI是什么?分为几种?怎么区分?(理解)

URI是统一资源标识符（英语：Uniform Resource Identifier) 用来标识互联网上某个资源名称的字符串.

而URI他又分为URL(定位符)和URN(名称), 他们三者的关系如下图所示:

本来对于资源来说, 有名称(URN)就够了, 比如用于标识唯一书目的ISBN系统, ISBN 0-486-27557-4 可以无独一无二地标识出莎士比亚的戏剧《罗密欧与朱丽叶》的特定版本, 去任何图书馆/任何书店输入ISBN就能查到这本书是否在馆内.

但是，后来人们发现, 网络资源, 除了知道资源名称, 还需要知道资源的位置, 即如何获取到这个资源, 这就衍生出了URL地址.

URN和URL有着互补的作用,一个好记,一个好找,就这么简单.
例如Google搜索其实就是一种知道资源名, 找资源地址的过程.

2. URL的格式是什么?

维基百科

URL标准格式(背下来)

协议类型:[//[访问资源需要的凭证信息可省略@]服务器地址[:端口号] ][/层级文件路径]文件名[?查询字符串][#锚点ID]

  1. 传输协议
  2. 层级URL标记符号([//]固定不变)
  3. 访问资源需要的凭证信息（可省略）
  4. 服务器。（通常为域名，有时为IP地址）
  5. 服务器接收端口号。（以数字方式表示，若为HTTP的默认值”:80”时, 可省略）
  6. 路径。（以“/”字符, 间隔路径中的每一个目录名称,到查询字符串为止）
  7. 查询字符串。（GET模式的窗体参数，以”?”字符为起点，每个参数以”&”隔开，再以”=”分开参数名称与数据，通常以UTF8的URL编码，避开字符冲突的问题）
  8. 锚点。以”#”字符为起点[2]

3. DNS英文全称是什么?

域名系统（英文：Domain Name System，缩写：DNS）是互联网的一项服务。

它由服务商提供, 作为将域名和IP地址相互映射的一个分布式数据库，可以让用户通过域名找到IP地址.

关于DNS有两个命令

nslookup www.baidu.com
ping www.baidu.com
输入这两个命令都会返回百度的IP地址

Server : 你的路由器地址,
接着, 路由器询问DNS服务商, DNS服务商询问百度后, 向你返回距离你最近的百度服务器的IP地址.

8.响应状态码都有哪类型?

(这小部分背下来)

状态吗就是服务器对浏览器(客户端)说的话：

  • 1xx 不常用
  • 2xx 成功
  • 3xx 滚到别的地方去
  • 4xx 你丫错了
  • 5xx 好吧我错了

其他的解释属于了解内容:**

200

通常的成功 OK

204

成功处理请求，但不会刷新页面
表示服务器接收到的请求已经处理完毕，但浏览器页面不会刷新

301 Moved Permanently

永久重定向
请求的网页已永久移动到新位置，永久性重定向

302 Found

HTTP/1.0就有的，临时性重定向，POST方法的重定向在未询问用户的情况下就会变成GET

303 和 307

这两个是HTTP/1.1新加的，都是临时重定向，303和302一样，POST重定向为GET。
307不同的是，把POST转为GET的。

304 Not Modified

表示自从上次请求后，网页未被修改过

400 Bad Request

客户端中存在语法错误。

401 Unauthorized

用户未授权，需要用户验证。

403 Forbidden

服务器已经理解请求，但拒绝执行

404 Not Found

服务器找不到请求的网页。

500 Internal Server Error

服务器遇到错误，无法完成请求。

503 Service Unavailable

由于临时的服务器维护或者过载，暂时无法处理请求

9. 如何使用Chrome查看请求和响应?

查看请求

  1. 打开 Network
  2. 地址栏输入网址
  3. 在 Network 点击，查看 request，点击「view source」
  4. 如果有请求的第四部分，那么在 FormData 或 Payload 里面可以看到.

    查看响应

  5. 打开Network
  6. 刷新网址
  7. 选中第一个响应
  8. 查看Headers里的Response Headers点view source可以看到响应的前两部分，Request Headers点view source可以看到请求
  9. 查看Response你可以看到响应的第四部分

  3. vim demo.txt

  4. 为demo添加可执行权限:chmod +x demo.txt

  5. cd到任何位置执行demo.txt这个脚本sh /Users/nvidintelamd/local/demo.txt，都可以在该位置生成一个demo项目。

应该注意的点

  2. 把/Users/nvidintelamd/local添加到~/.zshrc文件里的path里，可以免于输入local的绝对地址。也就是说，仅仅需要输入sh demo即可。

返回值+ && 的逻辑和用法

上述脚本在命令行里再执行一下 if脚本 && 2号命令, 会是一种什么逻辑呢?
简单来说就是”逢0执行”:
